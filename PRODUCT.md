# Product

## Register

product

## Users

Hackathon teammates, judges, and developers who need to understand how coding agents behave when repository content contains malicious or misleading instructions. They are usually scanning results quickly, comparing models, and looking for concrete evidence behind each verdict.

## Product Purpose

Trust Me Bro is an agent security benchmark dashboard. It helps users see which models safely handle hidden repo-level prompt injections, which scenarios expose risky behavior, and what evidence supports each outcome through canaries, traces, files, commands, and run summaries.

## Brand Personality

Minimal, data-forward, and trustworthy. The interface should feel closer to a serious benchmark lab than a marketing site: calm, crisp, and clear under time pressure.

## Anti-references

Avoid flashy cyberpunk security aesthetics, heavy gradients, decorative clutter, vague SaaS cards, and anything that makes the benchmark feel less credible. Do not obscure the data behind visual effects.

## Design Principles

- Lead with verdicts, then evidence.
- Make model comparison easy to scan.
- Treat canary failures as first-class security events.
- Keep dense information readable and calm.
- Use visual emphasis for status, severity, and selection rather than decoration.

## Accessibility & Inclusion

Aim for strong text contrast, keyboard-friendly controls, readable dense tables, and reduced-motion-safe interactions. Color should reinforce status but should not be the only way to understand a result.
